# v2 执行监控页面（桌面优先）页面设计

## 全局设计规范（适用于全部页面）

* Layout：桌面优先 12 栅格（CSS Grid + Flex 混合）。内容最大宽度 1200px，左右留白自适应；主内容区 Grid，卡片内用 Flex。

* Responsive：

  * ≥1200px：三列布局（左列表/中过程/右结果）

  * 768\~1199px：两列布局（上列表 + 下过程与结果分区）

  * <768px：单列堆叠（列表 → 时间线 → 快照 → 预测）

* Typography：

  * H1 20px/600，H2 16px/600，正文 14px/400，辅助文本 12px/400

* Colors（Token）：

  * \--bg: #0B1220（深色背景）

  * \--card: #0F1A2B（卡片底）

  * \--text: #E6EEF8；--muted: #9FB0C3

  * \--accent: #4F8CFF（主色）

  * \--success: #22C55E；--warning: #F59E0B；--danger: #EF4444

* Components：

  * Button：默认 accent 填充；hover 提亮 8%；disabled 降低不透明度 40%

  * Link：accent 文本，hover 下划线

  * Tag/Badge：用于状态（进行中/成功/失败/缺失）

  * Table：固定表头，行 hover 高亮；支持空态/加载态骨架屏

## 页面 1：首页（概览与入口）

### Meta Information

* title：足球预测｜v2 今日概览

* description：查看今天（12:00\~次日12:00）比赛、快照与 v2 预测产出概览。

* Open Graph：og:title 同 title；og:description 同 description

### Page Structure

* 顶部导航（横向）

* 主内容（单列卡片堆叠）：今日概览卡片 + 最近一次 v2 执行摘要 + 入口卡片

### Sections & Components

1. 顶部导航栏（Sticky）

   * 左侧：产品名/Logo

   * 右侧：菜单项（首页、v2 执行过程与结果）
2. 今日概览卡片（4 个 KPI）

   * 指标：当天比赛总数、已采集快照场次、已完成预测场次、异常场次

   * 角标：提示“当天=12:00\~次日12:00”
3. 最近一次 v2 执行摘要卡片

   * 行字段：开始时间、结束时间、状态 Badge、当前阶段（stage）、错误摘要（如有）
4. 入口卡片

   * 主按钮：进入“v2 执行过程与结果页”（/v2）

交互与状态：

* 加载态：KPI 与摘要显示骨架屏

* 空态：提示“暂无今天数据/尚未产生 v2 运行记录”

## 页面 2：v2 执行过程与结果页（核心）

### Meta Information

* title：v2 执行过程与结果｜当日比赛监控

* description：聚焦查看当天（12:00\~次日12:00）比赛数据、快照采集状态与最终预测结果。

* Open Graph：og:title 同 title；og:description 同 description

### Page Structure（桌面 ≥1200px）

* 三列布局（Grid 3 列）：

  * 左列：当天比赛列表（可筛选）

  * 中列：v2 执行过程时间线 + 数据一致性提示

  * 右列：快照状态与最终预测结果（分区卡片）

* 右侧抽屉：比赛详情（点击比赛行触发）

### Sections & Components

1. 顶部工具条（页面标题下）

   * 比赛日选择：DatePicker/下拉（默认“今天”），旁文固定说明“按 12:00\~次日12:00 归一”

   * 快速筛选：联赛下拉、关键字输入（队名/联赛）

   * 操作：刷新按钮（显示最后刷新时间）
2. 左列｜当天比赛列表卡片

   * Table 列：开赛时间、联赛、主队 vs 客队、比赛状态、快照状态 Badge、预测状态 Badge

   * 行点击：打开右侧详情抽屉

   * 辅助筛选：Tab（全部 / 未采集 / 预测失败）
3. 中列｜v2 执行过程时间线卡片

   * Timeline 节点：开始、拉取比赛、采集快照、生成预测、完成/失败

   * 每节点：状态（颜色）、时间、耗时、简短错误（如有）
4. 中列｜一致性提示卡片

   * 展示缺失统计：比赛总数、缺失快照场次、缺失预测场次

   * 若存在缺失：以 warning/danger 样式提示“当前数据不完整”
5. 右列｜快照采集状态卡片

   * Summary：已采集/未采集/失败数量

   * 明细 Table：比赛、snapshot\_type、状态、采集时间、失败原因（仅失败显示）
6. 右列｜最终预测结果卡片

   * Summary：已出/未出/失败数量

   * 明细 Table：比赛、model\_version、状态、生成时间、结果摘要（result 的关键字段按 v2 原样映射展示）
7. 右侧详情抽屉（宽 420\~520px）

   * 顶部：比赛基础信息（联赛、队伍、开赛时间）

   * 分区 1：快照明细（按 snapshot\_type 分组，含状态与错误）

   * 分区 2：预测明细（状态、生成时间、result JSON 关键字段；失败原因）

交互与状态：

* 加载态：三列区域独立骨架屏（避免全页阻塞）

* 空态：当日无比赛时显示“该比赛日无比赛数据”

* 错误态：拉取失败显示可读错误信息，并保留刷新按钮

* 视觉强调：

  * 未采集/预测失败：Badge 使用 danger/warning

  * 成功：success
