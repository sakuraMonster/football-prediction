# 进球数预测模型近期复盘与优化方案

## 一、 近期预测表现复盘 (2026-04-21 至 2026-04-23)
根据最近三天的 Excel 数据（共 35 场有效比赛）分析，进球数预测模型的准确率出现了明显下滑：
- **纯统计模型命中率**：17.14%
- **大模型基本面命中率**：31.43%

大模型（基本面预测）在 24 场比赛中预测失败。对这 24 场失败案例进行深度剖析，发现主要集中在以下两种极端错误模式：

### 1. 深盘高估陷阱 (预测 > 实际) —— “强队大热诱大”
- **典型失败案例**：
  - 巴黎圣曼 vs 南特 (盘口 3.5/4)：实际 3 球，预测 4, 5 球
  - 勒沃库森 vs 拜仁 (盘口 3.5/4)：实际 2 球，预测 3, 4 球
  - 伯恩利 vs 曼城 (盘口 3.5)：实际 1 球，预测 3, 4 球
  - 巴萨 vs 塞尔塔 (盘口 3.5/4)：实际 1 球，预测 3, 4 球
- **问题诊断**：当进球数盘口开到极深（≥3.5）且有豪门（曼城、拜仁、巴萨、大巴黎）参与时，大模型盲目相信了机构开出的“深盘”，默认进球数会达到 3-5 个。但实际上，豪门在密集赛程下往往采用“经济适用型”踢法（1-0, 2-0 即收兵），深盘往往是机构利用公众对豪门攻击力的盲目信任而设置的**“诱大”陷阱**。

### 2. 浅盘低估爆发 (预测 < 实际) —— “防线崩溃/对攻局”
- **典型失败案例**：
  - 国际米兰 vs 科莫 (盘口 2.5)：实际 5 球，预测 2, 3 球
  - 赫罗纳 vs 贝蒂斯 (盘口 2.5)：实际 5 球，预测 2 球
  - 埃尔切 vs 马竞 (盘口 2.5/3)：实际 5 球，预测 2, 3 球
  - 马尔默 vs 天狼星 (盘口 2.5/3)：实际 5 球，预测 2, 3 球
- **问题诊断**：当盘口仅为常规的 2.5 或 2.5/3 时，大模型过于受限于盘口，给出了保守的 2, 3 球预测。然而在实际比赛中，由于豪门虐菜（国米）、攻势足球碰撞（赫罗纳）或战意极端的保级战（埃尔切），一旦早早进球，防线极易崩溃，打出 4-5 球的大比分。大模型缺乏突破盘口限制、识别“潜在惨案”的能力。

### 3. 防守型联赛DNA被忽略（西甲、法乙、葡超）
- **典型案例**：
  - 毕尔巴鄂 vs 奥萨苏纳 (西甲)：实际 1 球，预测 3 球
  - 巴列卡诺 vs 西班牙人 (西甲)：实际 1 球，预测 2, 3 球
  - 卡萨皮亚 vs 布拉加 (葡超)：实际 1 球，预测 3, 4 球
- **问题诊断**：这些联赛（特别是中下游球队交手）天生具有很强的防守底色和较低的场均进球数。但大模型看到常规的 2.5 盘口时，容易给出偏高的预测（3球），完全忽略了球队本身沉闷的战术打法。

### 4. 大球联赛的“高开低走”冷门陷阱（荷甲、美职联）
- **典型案例**：
  - 前进之鹰 vs 阿尔克马 (荷甲)：实际 0 球，预测 2, 3 球
  - 盐湖城 vs 迈阿密国际 (美职联)：实际 2 球，预测 3, 4 球
- **问题诊断**：荷甲、美职联以“大球”著称，机构顺势开出 3 或 3.5 的深盘，公众理所当然认为会有进球大战。但如果遇到某一方近期进攻哑火或主力轮休，这就是典型的“诱大冷门”。大模型随大流预测了 3-4 球，掉入了陷阱。

## 二、 模型优化建议方案

针对以上复盘发现的致命盲点，建议对 `src/llm/goals_predictor.py` 的系统提示词（Prompt）和后处理逻辑进行以下升级：

### 1. 增加“深盘防诱规则 (Deep Handicap Trap)”
在 LLM 的 Prompt 中强制注入以下规则：
> **【深盘诱大预警】**：当进球数盘口达到或超过 `3.5`，且包含五大联赛顶级豪门（如曼城、拜仁、巴萨、巴黎等）时，这极大概率是机构的**诱大陷阱**。豪门在密集赛程下极易采用“经济适用”踢法。除非客队防线有极其严重的伤病，否则必须将预测核心放在 **1-3球**（小球方向），严禁盲目给出 4-5 球的预测。

### 2. 增加“浅盘防穿规则 (Shallow Handicap Explosion)”
在 LLM 的 Prompt 中强制注入以下规则：
> **【浅盘惨案预警】**：当进球数盘口仅为 `2.5` 或 `2.5/3`，但对阵双方存在以下情况时：（1）强队主场迎战弱旅且近期进攻极佳；（2）保级/争冠关键战中一方防线脆弱。此时不要被保守的盘口迷惑，极易打出单方面屠杀或对攻大战。请大胆将 **4-5球** 纳入核心预测选项。

### 3. 强化对“历史统计分布”的绝对服从
目前的 Prompt 虽然传入了历史统计数据，但大模型常常自作主张忽略它。需要增加强约束：
> **【统计分布约束】**：请严格参考提供的《进球数概率分布历史统计》。如果历史统计中该盘口下“小球（0-2球）”的概率超过 60%，你的预测**必须**包含 1球 或 2球，绝不允许与历史大概率背道而驰。

### 4. 引入“场均进球数”计算与“主力伤停”减档机制
1. **自动计算场均得失球**：在代码层提取爬虫抓取的近期战绩（如“30战18胜...进59失37”），计算出主客队的“场均进球”与“场均失球”，并明确打印在 Prompt 中作为数据锚点。
2. **伤停减档规则**：在 Prompt 中强制加入规则：
> **【伤停减档预警】**：当伤病名单中包含明确的主力前锋或中前场核心球员，或首发阵容残缺时，球队的进攻火力将大打折扣。此时你的进球数预期必须**强制下调1档**（即预期进球数减1）。

### 5. 补充联赛防守DNA与大球联赛冷门陷阱预警
在 Prompt 中加入以下联赛层面的补充判定规则：
> **【防守联赛DNA】**：西甲、法乙、葡超等天生防守型联赛，严禁盲目预测3球以上，即使盘口开到2.5也应优先防范1-2球。
> **【大球联赛冷门陷阱】**：荷甲、美职联等自带“大球属性”的联赛，若开出3/3.5深盘但基本面不支持（如主力伤停或近期进攻哑火），往往是机构利用公众固有思维的“高开低走诱大”陷阱，极易打出0-2球。

### 6. 核心记忆更新
建议使用 `manage_core_memory` 工具，将此次发现的“深盘诱大（3.5+）”、“浅盘击穿（2.5）”、“防守联赛DNA”和“大球联赛冷门”规则写入项目的 Core Memory，确保其他预测模型也能共享这一领域知识。

### 7. 引入高阶数据源：场均射门、场均进球与伤停情报（已完成融合）

**问题**：
基础的胜平负数据无法真实反映球队的进攻转化率和防守漏洞。

**优化方案（已实现）**：
1. **场均进失球提取**：已在数据融合和预测模型层（`predictor.py`）中，通过正则表达式自动从 500.com 的“近期战绩”字符串中计算出主客队的“场均进球”与“场均失球”。
2. **伤停情报提权**：已修改 `odds_crawler.py` 提取预计阵容中的“伤病”和“停赛”名单，并在 Prompt 格式化时独立作为一个核心特征块展示，确保大模型不会漏看核心前锋或后卫的缺阵。
3. **高阶数据爬虫架构（API-Sports）**：已建立 `AdvancedStatsCrawler` 类，支持通过配置 `FOOTBALL_API_KEY` 调用国际标准的 api-sports 接口，获取精准的赛季场均进球/失球数据。
4. **平滑降级机制（Fallback）**：若未配置 API Key 或遭遇反爬，系统将自动平滑降级，使用正则提取的 500网基础进失球数据进行替代，确保预测链路的稳定性。
