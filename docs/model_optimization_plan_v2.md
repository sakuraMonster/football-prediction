# 足球预测模型进阶优化方案 (基于 2026-04-12 复盘)

## 核心痛点回顾
1. **静态标签的局限性**：模型过去只懂得套用静态规则（如“看排名定强弱”、“西甲就是多平局”），但在赛季末，这些死板的规则常常被极端的保级战意所撕裂。
2. **“机构故事”的盲目信任**：当发现基本面（强队必胜）和盘口（浅盘无力）存在冲突时，大模型总是习惯性地用“阻上/诱下”来强行解释，而不愿意相信这是真正的冷门信号。
3. **“平胜”剧本滥用**：在北欧联赛或实力碾压局中，模型为了凑“平胜”的半全场玩法，强行假定上半场会僵持，无视了直接打出“胜胜”的巨大风险。

---

## 落地实施计划 (Action Plan)

为了让模型从一个“概率计算器”真正进化为一个“具备情景感知能力的风险管理者”，我们将分三步进行系统级改造：

### 第一步：引入“战意系数”与多维量化评估
**目标**：打破唯排名论，赋予大模型动态评估赛季末各支球队真实战意的能力。
**实施方案**：
1. **数据层扩展 (Data Layer)**：
   - 爬虫需增加获取“距离保级线/欧战区的积分差”、“近期5场抢分态势”、“关键球员伤病”等数据。
2. **Prompt 改造 (LLM Layer)**：
   - 在 `predictor.py` 中引入强制的 **战意乘数（0.8 - 1.2）** 概念。
   - **保级死磕（1.2）**：赛季末落后保级线 3 分以内的球队（尤其主场）。
   - **争冠/欧战冲刺（1.15）**：距离欧战区 3 分以内的球队。
   - **无欲无求/多线拖累（0.8）**：处于联赛中游且无保级压力的球队，或即将在下周中参加欧冠生死战的豪门。
   - 大模型在推演时，必须在报告中明确写出它为两支球队设定的“战意系数”，并将该系数直接作用于最终胜率的计算中。

### 第二步：实施分联赛的末期预测策略
**目标**：打破一刀切的静态标签，针对不同联赛的特性执行不同的过滤规则。
**实施方案**：
- **英超/德甲末期法则**：保级队主场爆发率极高。若中游强队（如热刺）客场对阵保级队，且让球无力，强制将主队不败概率上调。
- **西甲末期法则**：由于保级队和欧战队抢分极度凶狠，严禁大模型在涉及这些球队的比赛中滥用“平局”选项。
- **北欧联赛（瑞典超/挪超）法则**：对于豪门主场对阵中下游球队，系统必须默认优先评估“胜胜”（半场打穿）剧本。若无绝对理由，禁止推荐“平胜”。

### 第三步：推行“冷门背离”高赔预警策略
**目标**：当基本面和盘口发生剧烈冲突时，不再强行解释，而是拉响红色警报，提示博取高赔冷门。
**实施方案**：
1. **增加专属预警选项**：在 `predictor.py` 的输出格式中，除了常规的“竞彩推荐”外，新增一个 `🚨 高风险高回报冷门预警` 的独立输出项。
2. **触发条件强制化**：
   - **场景A（强队客场让球无力）**：强队客场面对弱队，基本面碾压，但盘口只开平手或平半。
   - **场景B（保级死磕不获支持）**：保级队主场生死战，战意爆棚，但盘口依然受让深盘。
3. **输出行为规范**：只要触发上述背离，模型必须在该栏目中明确写出：“本场数据与基本面严重背离，机构真实意图极可能是防冷。建议博取 [客队不胜 / 主队爆冷] 的高赔选项。”

### 第四步：“半全场”风险过滤器升级
**目标**：解决“平胜/平负”被滥用的问题。
**实施方案**：
- 修改 `htft_predictor.py`，加入**“胜胜”与“负负”的优先级拦截**。
- 大模型在考虑“平胜”前，必须先进行“胜胜”（即半场即被打穿）的证伪。如果强队战意满格且实力碾压，必须强制将“胜胜”作为首选，同时降低“平胜”的权重。如果在单关模式中无法买“胜胜”，则必须直接输出“建议放弃半全场玩法”。
