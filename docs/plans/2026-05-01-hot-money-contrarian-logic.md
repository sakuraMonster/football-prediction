# 热钱反向逻辑（Hot Money Contrarian）方案设计

## 一、用户提出的逻辑分析

### 用户原文
> 初赔代表机构的初始态度，后续的变化代表投入资金，当投入资金越大，赔率越低。
> 那么当某一方受助资金越多，赔率越低，而导致另外一方的赔率就会上升，
> 那么就代表赔率低的一方更受热，是否打出的概率就会变低
> （这时候就需要从初赔的角度看赔率高的一方初赔代表的实力）

### 逻辑拆解

这个逻辑可以拆成三层：

```
层级1：初赔 = 机构真实实力评估（无资金干扰时的纯净判断）
层级2：临场赔率变化 = 大众资金流向（谁被买得多，谁的赔率就降）
层级3：热门方（赔率下降方）≠ 必胜方，反而可能是"大热倒灶"
```

### 逻辑合理性判断

| 维度 | 评价 | 理由 |
|------|------|------|
| 初赔代表初始态度 | ✅ 合理 | 初赔是机构在未受市场资金影响时，基于纯实力、伤停、交锋等基本面开出的，是机构对比赛最底层的判断 |
| 赔率变化=资金流向 | ✅ 基本合理 | 欧赔的升降确实主要反映市场资金流向（受注量），买的人多→赔率降低→赔付风险减小 |
| 热门方反而危险 | ⚠️ 部分合理 | 这在足球博彩中被称为"大热倒灶"或"热钱陷阱"，但需要区分两种情况：① 真实热度（实力碾压导致的大热）vs ② 虚假热度（机构造热诱导导致的大热） |
| 冷门方初赔有参考价值 | ✅ 非常合理 | 如果初赔显示某队实力不差（比如初赔2.50），但临场因对手大热而被推高到3.50，这反而是价值洼地 |

### 补充修正（让逻辑更严谨）

用户逻辑的核心是正确的，但需要补充一个关键区分：

```
❌ 简单粗暴：赔率下降 = 大热 = 不赢
✅ 精准版本：赔率下降幅度 > 初赔隐含实力应有的范围 = 过热陷阱 = 不赢
```

**核心量化指标：欧赔变异系数 = (临赔 - 初赔) / 初赔**

- 如果主胜赔从 2.00 降到 1.80（降幅 10%），但初赔 2.00 本身就已经表达了主队优势 → 这是**实力碾压型热度**，主队大概率赢
- 如果主胜赔从 2.80 降到 2.30（降幅 18%），但初赔 2.80 说明主队实力并不占优 → 这是**虚假过热**，机构可能在造热诱导

## 二、落地实现方案

### 方案概述

在现有的"动态风控规则引擎"基础上，新增一个**"欧赔热钱反向检测模块"**，自动识别"某一方初赔→临赔发生显著下降"的过热陷阱。

### 实施步骤

#### Step 1：扩展规则引擎的上下文变量

在 `predictor.py` 的 `_build_rule_context()` 中新增欧赔维度变量：

```python
context["euro"] = {
    "home_init": 2.50,      # 主胜初赔
    "home_live": 2.10,      # 主胜临赔
    "draw_init": 3.20,
    "draw_live": 3.40,
    "away_init": 2.80,
    "away_live": 3.50,
    
    # 衍生变量
    "home_drop_pct": -0.16,        # (2.10 - 2.50)/2.50 = -16%
    "home_is_hot": True,            # 降幅>5%则为热
    "away_rise_pct": +0.25,         # (3.50 - 2.80)/2.80 = +25%
    "max_drop_side": "home",        # 降幅最大的一方
    "max_money_flow_side": "home",  # 资金流向最大方
    
    # 关键：初赔隐含实力
    "home_init_strength": "moderate",  # 2.00-2.50=较强, 2.50-3.00=一般, >3.00=弱
    "away_init_strength": "weak"
}
```

#### Step 2：在 JSON 规则库中新增热钱反向规则

在 `data/rules/micro_signals.json` 中追加以下规则：

```json
{
  "rule_id": "hot_money_contrarian_general",
  "name": "热钱反向：赔率骤降方无胜",
  "category": "micro_signal",
  "level": "🔴高危",
  "condition": "euro['max_drop_pct'] >= 0.05 and euro['max_drop_side_init_strength'] == 'weak'",
  "warning_template": "【热钱反向陷阱】{euro['max_drop_side']}方赔率从初赔 {euro['max_drop_side_init']} 骤降至临赔 {euro['max_drop_side_live']}（降幅 {euro['max_drop_pct']:.1%}），但该方初赔隐含实力仅为'{euro['max_drop_side_init_strength']}'级别。资金热度远超其真实实力，属于经典的虚假过热/造热诱导陷阱。根据热钱反向逻辑，该方基本无胜，应坚决走反向（冷门方不败甚至直接赢球）。",
  "retry_directive": "不让球推荐必须排除{max_drop_side}胜，重点关注冷门方向！"
}
```

#### Step 3：更新 Prompt 风控规则

在 `rules.py` 的 `DYNAMIC_CHANGE_RULES` 中追加：

```python
# 热钱反向通用规则
HOT_MONEY_CONTRARIAN_RULE = """
- **【热钱反向铁律】**：当某一方的初赔→临赔降幅超过5%，但该方初赔本身并未体现明显实力优势（初赔>2.30），则判定为"虚假过热陷阱"。此时该热门方基本无胜，应坚决走反向（冷门方不败/赢球）。
"""
```

#### Step 4：Streamlit 规则管理面板可视化

在 `5_Rule_Manager.py` 中新增一个"热钱反向规则"卡片区域，展示：
- 当前所有比赛的热力度排行（哪场资金最热）
- 触发过热预警的比赛高亮显示

## 四、总结

**用户逻辑的核心是正确的，且与博彩圈公认的"大热倒灶"理论一致。**

关键优化点在于：**不能只看赔率变化幅度，必须结合初赔隐含实力来区分"实力型热度"与"虚假过热"**。

这套逻辑落地后，将与现有的"盘赔微观信号引擎"形成互补：
- 盘赔微观信号 → 从亚盘盘口变化识破机构手法
- 热钱反向逻辑 → 从欧赔资金流向识破大众陷阱
