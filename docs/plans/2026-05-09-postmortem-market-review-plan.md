# 05-08 盘口复盘优化与规则回灌计划

**目标**

把 2026-05-08 复盘页从“总结报告”升级为“盘口规则迭代入口”。本轮重点不是补样例，而是解决 `盘口复盘缺失`、`已有信号适用边界未复盘`、`无信号场次无法沉淀新规则` 三类问题。

**本轮聚焦的 4 场**

1. `周五005 | 埃夫斯堡 vs 布鲁马波`
   - 现象：命中 `half_ball_water_drop_no_rise`，但 Agent C 跟随盘赔后误判。
   - 目标：判断该规则是完全背离，还是边界过宽；输出“优化旧规则 / 拆分子规则”结论。
2. `周五003 | 帕德博恩 vs 卡斯鲁厄`
   - 现象：当前系统缺少有效规则覆盖。
   - 目标：产出“新增规则候选”而不是只做泛化总结。
3. `周五009 | 都灵 vs 萨索洛`
   - 现象：与周五005同属盘口误读，但不是同一盘型。
   - 目标：判断是优化旧规则还是新增独立 guard。
4. `周五012 | 莱万特 vs 奥萨苏纳`
   - 现象：复盘正文只有简短结论，缺少完整盘口链路。
   - 目标：把“盘口复盘完整性”变成硬约束，杜绝漏写。

---

## 一、问题重定义

当前复盘链路已能输出：

- 四维方向对比
- Agent C 误判链路
- 盘口错因总结
- 候选规则草稿

但仍存在三个结构性缺陷：

1. **逐场盘口复盘不完整**
   - 错误比赛可能只被一句话概括，没有完整的盘口链路。
   - `周五012` 就属于这一类。
2. **规则处置不够明确**
   - 已命中的规则误判后，报告没有强制回答：
     - 是旧规则完全错误？
     - 还是适用边界不清？
     - 应该优化、拆分还是停用？
3. **无规则覆盖场次缺少标准出口**
   - 像 `周五003` 这类场次，应该明确走“新增规则候选”路径，而不是停留在文本洞察。

---

## 二、改造目标

### 第一阶段：补“逐场盘口复盘完整性”

每个错误场次必须输出以下 4 个要素：

1. `盘口链路`
   - 初盘、即时盘、水位变化、盘水关系、欧亚联动
2. `现有规则命中检查`
   - 命中了哪些规则
   - 哪条规则主导了误判
3. `规则处置建议`
   - `optimize_existing`
   - `add_new_rule`
   - `observe_only`
4. `可执行结论`
   - 若可立规，必须给草稿
   - 若暂不立规，也必须说明原因

### 第二阶段：补“规则处置分类”

候选规则草稿新增分类字段：

- `disposition`
  - `optimize_existing`
  - `add_new_rule`
  - `observe_only`
- `based_on_rule_id`
  - 若是优化旧规则，写明对应 rule id
- `market_review_complete`
  - 标记该草稿是否基于完整盘口复盘得出

### 第三阶段：补“复盘完整性校验”

保存复盘前，程序化校验：

1. 每个错误场次是否都在正文中被点名
2. 每个错误场次是否出现“盘口链路/规则命中/处置建议”之一
3. 若缺失，生成“盘口复盘缺失”警告

---

## 三、对四场比赛的处置建议

### 周五005 | 埃夫斯堡 vs 布鲁马波

建议归类：`optimize_existing`

- 目标规则：`half_ball_water_drop_no_rise`
- 判断：
  - 不建议直接废弃
  - 更像是适用边界过宽
- 方向：
  - 收紧旧规则适用条件
  - 必要时拆出“半球盘超低水控热保护”子规则

### 周五003 | 帕德博恩 vs 卡斯鲁厄

建议归类：`add_new_rule`

- 目标：新增“深盘升盘升水预警”
- 理由：
  - 这是现有规则库覆盖空白
  - 不应强行塞进现有半球盘规则

### 周五009 | 都灵 vs 萨索洛

建议归类：`add_new_rule`

- 目标：新增“平半退平手低水正路辩护”或对应仲裁 guard
- 理由：
  - 与周五005不是同一盘型
  - 更像独立的“退盘保护”剧本

### 周五012 | 莱万特 vs 奥萨苏纳

建议归类：`observe_only` 或 `missing_market_review`

- 当前主要问题不是立即立规，而是盘口复盘缺失
- 先修流程，确保以后这类场次必有盘口链路

3. `half_ball_water_drop_no_rise` 相关误判不再只能落成泛化总结，而是能形成明确处置建议

   - Prompt 含逐场复盘与处置分类要求
   - `validate_review()` 能识别盘口复盘缺失
   - 规则草稿新增字段可被提取并展示
