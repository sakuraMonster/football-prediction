# 昨日赛果复盘与预测模型优化计划

## 1. 目标
拉取昨日（2026-04-04，星期六/周五赛事）的实际比赛结果，与数据库中保存的系统预测结果进行对比，统计预测准确率，并利用大语言模型（LLM）对错误预测的场次进行深度归因分析，最终提出针对性的模型/提示词优化方案。

## 2. 实施步骤

### 步骤一：拉取昨日实际赛果
- 利用现有的 `JingcaiCrawler.fetch_match_results(target_date)` 方法，传入昨日的日期（如 `2026-04-04`，注意周五赛事可能跨越到凌晨）。
- 获取所有“周五”开头（或对应日期）赛事的实际比分。

### 步骤二：获取系统历史预测记录
- 查询 `Database` 中保存的昨日 `MatchPrediction` 记录。
- 将预测的“竞彩推荐”（胜平负/让球胜平负）与抓取到的“实际比分”进行匹配对比。
- **比对逻辑**：
  - 根据主客队进球数计算出实际的胜平负。
  - 结合保存的让球数值（如 `-1`），计算出实际的让球胜平负。
  - 判断 LLM 当时的预测结果是否命中。

### 步骤三：统计准确率与错误归因
- 汇总统计：总场次数、命中场数、错误场数、整体胜率。
- **针对错误场次**：
  - 提取当时传递给 LLM 的原始盘口数据、基本面信息以及 LLM 给出的预测理由。
  - 结合真实赛果，将这些信息组合成一段反思 Prompt，重新提交给 LLM。
  - 要求 LLM 进行“赛后复盘分析”，指出当时分析的盲点在哪里（例如：是否被诱盘？是否低估了伤停影响？是否对战意判断错误？）。

### 步骤四：输出改进方案
- 收集 LLM 给出的复盘反思。
- 总结归纳导致预测失败的核心共性原因。
- 提出具体的系统改进方案（例如：调整 `predictor.py` 中的提示词侧重点、增加新的基本面数据源、调整置信度算法等）。
